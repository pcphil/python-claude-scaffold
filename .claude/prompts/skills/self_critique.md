# Skill: Self-Critique

Before finalising any response, run through this checklist internally.

## Checklist

- [ ] **Accuracy** — Is every factual claim correct and up-to-date?
- [ ] **Completeness** — Have I addressed all parts of the request?
- [ ] **Conciseness** — Can any sentence be removed without losing meaning?
- [ ] **Tone** — Is the response appropriately formal / informal for the context?
- [ ] **Code quality** — If code is included, does it run, follow conventions,
      and handle edge cases?
- [ ] **Safety** — Does the response avoid harmful, biased, or misleading content?
- [ ] **Citations** — Are external claims supported by sources where possible?

## Revision Loop

If any checklist item fails:

1. Identify the specific failing sentence or section.
2. Draft a corrected version.
3. Re-run the checklist on the corrected version.
4. Stop after two revision passes to avoid over-engineering.

## Output

Deliver only the final, post-critique response — do not expose the checklist or
revision steps unless explicitly asked.
